Sellerhub 구조 및 개발 코드 감사 보고서

현재 저장소는 대형 PHP 모놀리식 코드베이스이며, 공통 부트스트랩 과결합, 디렉터리 경계 붕괴, 실운영 의존 테스트, 민감정보 노출 흔적, 레거시 문법 혼재가 동시에 존재합니다. 이번 보고서는 실제 파일 탐색 결과를 기반으로, 유지보수성과 안정성을 가장 크게 훼손하는 지점을 우선순위 중심으로 정리합니다.

프로젝트 개요

탐색 기준은 저장소 내부 파일 구조, 대표 엔트리포인트, 공통 부트스트랩, 테스트 구성, 위험 신호 패턴 검색 결과입니다.

요약 판단

  • 핵심 리스크는 개별 버그보다도 시스템 경계가 없는 구조에 있습니다.
  • shop/lib/library.php 하나에 HTTP 헤더, 인코딩, dotenv, 세션, Redis, DB, Eloquent, 디버그 설정이 집중되어 있습니다.
  • shop/, m/, m2/, whub/가 독립 서비스처럼 보이지만 실제로는 공통 전역 상태를 공유합니다.
  • 테스트는 존재하지만 상당수가 실DB, 실데이터, 외부 웹훅에 의존해 회귀 안전망으로 보기 어렵습니다.
  • 레거시 PHP 문법, EUC-KR 헤더, short tag, 직접 include, 셸 실행이 광범위하게 남아 있어 현대화 비용이 누적되고 있습니다.

  • 공통 부트스트랩: shop/lib/library.php
  • 테스트 구성: phpunit.xml, tests/bootstrap.php, tests/ 전반

상세 이슈

각 이슈는 심각도, 영향, 근거, 개선 방향을 함께 기록했습니다. 단순 취향 문제가 아니라 구조적 비용을 만드는 항목만 포함했습니다.

공통 부트스트랩에 런타임 책임이 과도하게 집중됨

shop/lib/library.php가 전역 애플리케이션 컨테이너처럼 동작하면서, HTTP 응답 제어와 인프라 초기화, 세션, ORM 로딩이 한 파일에 결합되어 있습니다.

CRITICAL
영향: 변경 파급도 높음 범주: 구조 대표 파일: shop/lib/library.php
  • 웹 요청 헤더 출력, 인코딩 고정, dotenv 로딩, 세션 쿠키 정책, Redis 세션 핸들러, DB, Eloquent, 디버그바 설정이 모두 여기서 수행됩니다.
  • 이 구조에서는 특정 하위 기능만 테스트하거나 CLI/배치/웹 환경을 분리하기 어렵습니다.
  • 부트스트랩 수정이 인증, 페이지 렌더링, 배치, 테스트 전체에 동시에 영향을 줍니다.
shop/lib/library.php
header("Content-Type: text/html; charset=EUC-KR");
$dotenv = Dotenv::createImmutable($_SERVER['DOCUMENT_ROOT']);
session_start();
require_once $_SERVER['DOCUMENT_ROOT'] . '/shop/lib/eloquent.php';
require_once $_SERVER['DOCUMENT_ROOT'] . "/shop/lib/sh/sh.class.php";

권장 방향: HTTP/CLI 공통 초기화, 세션 초기화, 데이터 계층 부트스트랩을 분리하고 엔트리포인트별 최소 부트스트랩으로 축소해야 합니다.

민감정보와 외부 웹훅이 코드 및 테스트에 직접 박혀 있음

실서비스용 추정 Slack/Synology 웹훅, 외부 API 키, 시크릿이 코드와 테스트에 노출되어 있습니다.

CRITICAL
영향: 보안 사고 가능성 범주: 보안/운영 대표 파일: tests/SynologyTest.php, shop/lib/sh/ProviderMgr.php
  • 테스트 파일 안에 외부 웹훅 URL이 존재하며, 실제로 요청을 보내는 코드가 있습니다.
  • 운영 코드 곳곳에도 Slack/Synology 훅 URL과 키 관련 필드가 직접 나타납니다.
  • 이 패턴은 저장소 유출, 로그 노출, 개발자 로컬 실행만으로도 외부 시스템에 영향이 갈 수 있음을 의미합니다.

테스트가 격리되지 않아 회귀 안전망 역할을 못 함

phpunit.xml은 존재하지만 tests/bootstrap.php가 실제 애플리케이션 부트스트랩을 그대로 올리고, 테스트 코드도 실DB/실데이터/외부 시스템을 직접 사용합니다.
